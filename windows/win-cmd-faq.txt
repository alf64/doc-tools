########## Using pnputil - utility for managing device drivers ##########
Using pnputil to install driver:
pnputil /add-driver <driver-file.inf> /install /force /reboot

List the drivers installed on the system:
pnputil /enum-drivers

Deleting driver
pnputil /delete-driver <driver-file-obtained-fromenum-drivers.inf> /uninstall /force /reboot

########## Configure Windows 10 (and 11) to act as NTP (Ntwork Time Protocol) server ##########
Open the command prompt (cmd) as administrator and execute commands described below.
    // Stop the Windows time service for the purpose of reconfiguration:
    net stop W32Time

    // Reconfigure the service
    reg add “HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\W32Time\Config” /v LocalClockDispersion /t REG_DWORD /d 0 /f
    reg add “HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\W32Time\Parameters” /v LocalNTP /t REG_DWORD /d 1 /f
    reg add “HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\W32Time\TimeProviders\NtpServer” /v Enabled /t REG_DWORD /d 1 /f
    reg add “HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\W32Time\Config” /v AnnounceFlags /t REG_DWORD /d 5 /f

    // Set service auto-start
    sc config W32Time start=auto

    // Start the Windows time service
    net start W32Time

    // Check if the NTP server is enabled:
    w32tm /query /configuration
    // Check the 2nd to last line in the output; Enabled: 1 (local) is correct.

    // Add a rule to the firewall to allow this service:
    netsh advfirewall firewall add rule name=”NTP Server” dir=in protocol=udp localport=123 profile=any enable=yes action=allow

    // You can test the service from an external computer that runs Windows and is connected to the network with the server, with the following command:
    w32tm /stripchart /computer:192.168.0.10 /samples:4 /dataonly
    // (replace the IP address with that of the computer that now runs the NTP Server)

########## Other commands & hints ##########
--- Display file tree of a folder
tree <name-of-the-folder> /F

--- List current directory insights
dir

--- Change disk drive to D:\
D:

--- Change directory (path-to-directory must be on the disk drive you are currently on, i.e. if you want to cd to D:\some-folder you need to change disk to D:\ first)
cd <path-to-directory>

--- Delete currently written command
'Ctrl + Backspace'

--- Where the useful Windows apps are that you can run using cmd.exe or through Run prompt (Win+R)
They are in C:\Windows\System32\

--- Open classic 'Control Panel' in Windows 10/11:
Hit Win+R which will open small 'Run...' window.
In the window type: appwiz.cpl and hit Enter
Control Panel will open, listing installed apps. Now you can navigate it freely and dive into all the options offered by it.
Alternatively you can call appwiz.cpl from cmd.exe

--- Open 'Disk Management' in Windows 10/11:
Hit Win+R which will open small 'Run...' window.
In the window type: diskmgmt.msc and hit Enter
Disk Management window app will open, allowing you to manage disks (delete & create partitions).
Alternatively you can call diskmgmt.msc from cmd.exe (cmd.exe needs to be opened as Administrator).

--- Install Windows features from .cab files
If you have a .cab file that contains Windows feature, you can install it using the following command:
DISM /Online /Enable-Feature /FeatureName:NetFx3 /All /Source:D:\sources\sxs /LimitAccess
Where:
 - /Source argument is a path where your .cab file is (in this example: D:\sources\sxs).
 - /FeatureName is a part of a .cab filename. Example filename: Microsoft-Windows-NetFx3-OnDemand-Package~31bf3856ad364e35~amd64~~.cab

--- Change default Terminal from PowerShell into Command Prompt (cmd.exe):
 - right click anywhere in File Explorer and select 'Open in Terminal'
 - by default PowerShell will open
 - in PowerShell, right click on its name bar (on the top of the window) and select 'Settings'
 - Startup settings will open, in the first 'Default profile' setting change 'PowerShell' into 'Command Prompt'. Hit Save button
 - from now on right clicking anywhere in File Explorer and selecting 'Open in Terminal' will open Command Prompt instead of PowerShell

--- Where the apps installed thorugh MS Store are:
C:\Users\username\AppData\Local\Microsoft\WindowsApps\

--- How to call the 'Set Environmental Variables' window from cmd:
rundll32 sysdm.cpl,EditEnvironmentVariables

--- Desktop shorcuts handling in .cmd ---
You may see shortcuts on your Desktop, but when you cd to the C:\Users\<username>\Desktop and inspect (command: dir) what's there you can
find out that they are in fact not there.
The reason is that they may reside in public desktop, which is: C:\Users\Public\Desktop (this dir is hidden in file explorer)
Things at public desktop are visible in all users desktop, as they usually contains shortcuts for apps installed with 'for all users' option.
You may delete these shortcuts in public desktop and they should disappear from your desktop.

--- Smart App Control ---
Windows 11 introduced Smart App Control ('Inteligentna kontrola aplikacji'), which can block your apps when it decides that they are dangerous.
What's worse you may not even know that your app has been blocked.
I had a trouble installing MS Office 2013 - the installer was giving me a general error during installation.
I switched the Smart App Control off and it helped. You can switch it off by doing the following:
Ustawienia -> Prywatnosc i zabezpieczenia -> Zabezpieczenia Windows -> Kontrola aplikacji i przegladarki -> Inteligentna kontrola aplikacji -> Wylacz

########## Managing network routing table ##########
# All commands referenced below are to be used in cmd.exe.
# Assumption: computer's IP is static and exist in 192.168.21 network, i.e. computer's IP is 192.168.21.3
# In the same network there exist another device of IP 192.168.21.211 and this device acts a gateway to another network (10.1.1)

# Displaying routing table for IPv4:
route print -4
# The priority of routing is most probably from bottom to top.
# This means that firstly the rule at the bottom is rspected, then if it does not resolve, the next one above is taken into account, etc.

# Adding new routing rule:
route add <destination-ip> MASK <subnet-mask> <gateway-ip>

# Add new routing rule - example for 10.1.1 'outside network':
route add 10.1.1.0 MASK 255.255.255.0 <gateway-ip>
# example command:
# route add 10.1.1.0 MASK 255.255.255.0 192.168.21.211
# Above command means that any access to/from network 10.1.1 must come to/from (or simply: through) 192.168.21.211 (because the 192.168.21.211 acts as a gateway in your own network)
# I.e. when on your computer you do 'ping 10.1.1.11', the packets will be send to the device with IP: 192.168.21.211 and this device should pass it to target
# device with IP: 10.1.1.11 (target device exists in another network - 10.1.1 - and must be reached through the gateway).
# Same goes for receiving packets from target device - they will be received through the gateway.

# Added routing rule exists only until computer's power off.
# If you want to make the rule persistent, use the '-p' flag when you do 'route add' command.

# Routing rules can be deleted using the following command:
route delete <destination-ip>
# example command:
# route delete 10.1.1.0
